Bygg din portfölj

I bästa fall har Juniora Uvecklare max ett par skolprojekt på GitHub. Att ha en välluppgydd portfölj kan vara en stor fördel när man söker sitt första jobb. För det första är det en chans to stand out from the crowd. För den andra tas kandidaten på allvar när man kan se hens riktiga projekt och engagemang. En aktiv uppdaterat GitHub med avancerade och interesanta projekt kan x10 chanser för interview. Egna projek säger ttill arbetsgivaren att personen kan programmera, skapa avancerade och långsiktiga projekt Här är de några tips hur man bygger en bra GitHub porfölj: publicera/spara alla skolprojekt till GitHub Arbeta med dina egna hobby projekt. Gärna långsiktiga som går att utveckla och bygga på under flera år Fokusera på sådana projekt som är attraktiva för riktiga besökere och eventuella kunder. Detta är bra tecken och säger att du kan leverera en färdig produkt som uppskattas av folk Gör collaboration. Kan vara dina arbetskamrater, kollegor eller andra juniora utvecklare som du kan hitta på t.ex. på Linkedin Delta i open source projekt I bästa scenario ska du undvika kopior på andras appar, super enkla appar/populära appar som alla har (t ex kalkylator), appar från kurser. Det är fortfarande bra att publicera de för att bygga mängden samt att träna oh uttvecklas regelbundet men i fronten ska det finnas seriösa appar. Välj 6 bästa/starka projekt och pin them på din GitHub Fyll i annat info på din github profil Googla gärna annat info hur kan man göra din GitHub attraktiv Har du jobbat med projekt som du får inte/går inte publicera på github till ex från ett annat arbetsplats eller praktik så är det också bra att berätta om dessa projekt i din detaljerat CV. Beskriva vad du har gjort, vilka teknologier du använde etc Försök uppdatera dina projekt regelbundet. Inget kan berätta mer om utvecklare än kalender som ser ut så här. Det säger att du är seriös, konsistent och kan visa mer än de flesta kandidater vilket betyder att det är dags att kalla dig för en inervju och lära känna dig.

Visa upp dina projekt och kod. Lägg in GitHub-länkar, dokumentation och korta beskrivningar.
